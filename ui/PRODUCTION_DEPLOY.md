# Production Deployment Guide

## Overview
This application consists of:
1. **MongoDB API Backend** - Express server that connects to MongoDB Atlas
2. **React UI** - Static frontend built with Vite

## Quick Deploy to Render

### Prerequisites
- GitHub account
- Render account (free at render.com)
- MongoDB Atlas cluster (already configured)

### Deployment Steps

1. **Push Code to GitHub**
   ```bash
   git init
   git add .
   git commit -m "Production ready deployment"
   git remote add origin YOUR_GITHUB_REPO_URL
   git push -u origin main
   ```

2. **Deploy on Render**
   - Go to https://render.com/dashboard
   - Click "New +" and select "Blueprint"
   - Connect your GitHub repository
   - Select the repository with this code
   - Render will automatically detect `render.yaml` and create both services:
     - `insurematrix-api` (Backend API)
     - `insurematrix-ui` (Frontend)

3. **Verify Deployment**
   - Backend health check: `https://insurematrix-api.onrender.com/mongo-api/health`
   - Frontend: `https://insurematrix-ui.onrender.com`

### Environment Variables
Already configured in `render.yaml`:
- `VITE_MONGO_URI` - MongoDB connection string
- `VITE_MONGO_DB` - Database name (instest)
- `VITE_MONGO_COLL` - Collection name (results)
- `PORT` - Auto-generated by Render

### Manual Deployment (Alternative)

If you prefer manual setup:

1. **Deploy Backend**
   - New Web Service
   - Build Command: `npm install`
   - Start Command: `node mongo-api.cjs`
   - Add environment variables from `.env`

2. **Deploy Frontend**
   - New Static Site
   - Build Command: `npm install && npm run build`
   - Publish Directory: `dist`
